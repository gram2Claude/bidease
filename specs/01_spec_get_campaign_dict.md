# Spec for get_campaign_dict

## Summary

Новая публичная функция в `bidease.py`, возвращающая **справочник кампаний** аккаунта
(DataFrame: одна строка на кампанию).

**Особая семантика (специфика Bidease):** справочников-эндпоинтов в API нет — справочник
собирается из **единственного stats-эндпоинта** группировками по атрибутам кампании
(`CampaignID`, `CampaignName`, `AdvertiserID`, `ProductID`) за максимально доступный период
(последний год). Метрики из ответа отбрасываются. В справочник попадают только кампании,
имевшие хотя бы одно событие за период, — «неактивные с рождения» кампании API не отдаёт.

## Functional Requirements

### 1. Сигнатура функции

```python
def get_campaign_dict() -> pd.DataFrame:
```

Без параметров (зеркально avito). Период запроса вычисляется внутри.

### 2. Алгоритм

**[SYNC] Синхронный паттерн:**

1. Вычислить период локально (относительные даты API не используем — конвенция проекта):
   - `fromdate = сегодня − 364 дня` (безопасно внутри лимита «1 год от текущей даты»;
     366 дней и больше → HTTP 400, установлено фактом);
   - `todate = сегодня` (эксклюзивна → данные по вчера включительно; кампания, впервые
     запущенная сегодня, появится в справочнике завтра — допустимо для справочника).
2. `GET /api/reporting/v1/stats` с параметрами (повторяемый `group` — списком пар):
   `api-token`, `fromdate`, `todate`,
   `group=CampaignID`, `group=CampaignName`, `group=AdvertiserID`, `group=ProductID`.
3. Ответ — CSV: 35 колонок метрик, затем колонки группировок **в нижнем регистре
   в порядке запроса**: `campaignid`, `campaignname`, `advertiserid`, `productid` (факт).
4. Оставить только колонки группировок, переименовать в snake_case:
   `campaign_id ← campaignid`, `campaign_name ← campaignname`,
   `advertiser_id ← advertiserid`, `product_id ← productid`.
5. Отбросить строки без `campaign_id`; дедупликация по `campaign_id`
   (`keep="first"`, как в avito).
6. Обогащение и приведение к фиксированному набору/порядку колонок
   `CAMPAIGN_DICT_COLUMNS` (раздел 3).

### 3. Возвращаемый DataFrame

Все имена колонок — **snake_case**. Порядок — `CAMPAIGN_DICT_COLUMNS`:

| Колонка | Тип | Описание |
|---------|-----|----------|
| `campaign_id` | integer | ID кампании (CSV `campaignid`) |
| `campaign_name` | string | название кампании (CSV `campaignname`) |
| `advertiser_id` | integer | ID рекламодателя на ui.bidease.com (CSV `advertiserid`) |
| `account_id` | integer | константа `1` (пример, заменить при интеграции) |
| `source_type_id` | integer | константа `10` (решение проекта 2026-07-21) |
| `product_id` | integer | **реальный `ProductID` из API** (CSV `productid`) — ⚠️ отступление от avito-конвенции, где это константа `1`; решение зафиксировано 2026-07-21 |
| `product_name` | string | константа `"prod_test"` (пример) |
| `camp_type` | string | константа `"camp_test"` (пример) |
| `camp_category` | string | константа `"cat_test"` (пример) |
| `id_key_camp` | string | вычисляется: `"1_" + campaign_id` |
| `owner_id` | integer | константа `1` (пример) |

#### Обязательное обогащение DataFrame (соглашение проекта)

Применён вариант «справочник» с проектными значениями Bidease:
`account_id = 1`, `source_type_id = 10`, `product_name`/`camp_type`/`camp_category` —
константы-примеры, `owner_id = 1`, `id_key_camp = "1_" + campaign_id`.
`product_id` — из API (см. таблицу выше). Расходов в справочнике нет —
блок `costs_*`/`ak` не применяется.

### 4. Изменения в `bidease.py`

- Реализовать тело `get_campaign_dict()` (заменить стаб `NotImplementedError`).
- Добавить приватный хелпер `_dict_period()` (вычисление `fromdate`/`todate` справочника)
  либо инлайн — на усмотрение плана.
- Маппинг CSV-колонок группировок → snake_case — модульная константа
  (переиспользуется следующими функциями).
- Обновить docstring модуля (функция реализована).

## Ограничения API

| Ограничение | Значение | Реализация |
|-------------|----------|------------|
| Период | в пределах 1 года от текущей даты | `fromdate = сегодня − 364d` |
| `todate` | эксклюзивна, строго > `fromdate` | `todate = сегодня` |
| Группировок в запросе | ≤ 7 | используем 4 |
| Rate limits | не документированы | 429-backoff в `_get_report` (защитный) |
| Ошибки | 401 пустое тело; 400 plain-text | `raise_for_status` → `requests.HTTPError` |

**Поведение при 429:** экспоненциальный backoff — база 1 сек, удвоение, макс. 5 повторов,
уважается `Retry-After` (уже реализовано в `_get_report`).

## Possible Edge Cases

- **Пустой аккаунт** (текущее состояние токена проекта — данных нет за весь год):
  ответ с группировками = только строка заголовка → `_parse_csv` даёт DataFrame без строк →
  вернуть `pd.DataFrame(columns=CAMPAIGN_DICT_COLUMNS)`. Не ошибка.
- **Кампания переименована в течение года:** если API отдаёт имя историческим срезом,
  один `campaign_id` даст 2+ строки с разными именами — дедупликация оставит первую
  (см. Open Questions).
- **Пустое/отсутствующее `productid` или `advertiserid`** в отдельных строках →
  значение `NULL` (NaN), строка не отбрасывается (отбрасываем только без `campaign_id`).
- **Целочисленные ID в пустом DataFrame** → dtype не приводить насильно при 0 строк
  (пустой фрейм с правильными именами колонок достаточен; типы фиксируются на данных).

## Acceptance Criteria

- [ ] `get_campaign_dict()` возвращает DataFrame ровно с колонками `CAMPAIGN_DICT_COLUMNS`
      в заданном порядке.
- [ ] Одна строка на уникальный `campaign_id` (дедупликация работает); строк без
      `campaign_id` нет.
- [ ] `id_key_camp == "1_" + str(campaign_id)` для всех строк; `source_type_id == 10`.
- [ ] При отсутствии данных (пустой аккаунт) возвращается пустой DataFrame с правильными
      колонками — без исключений.
- [ ] Unit-тесты (pytest, мок HTTP): нормальный CSV, пустой CSV (только заголовок),
      совсем пустое тело, дубль `campaign_id`, строка без `campaign_id`.
- [ ] Smoke-тест `bidease/smoke_tests/test_get_campaign_dict.py` создан и выполнен на
      живом API; результат сохранён в `bidease/raw_data/get_campaign_dict.csv` (cp1251),
      показаны первые 5 строк из CSV (с текущим токеном допустим корректный пустой результат).
- [ ] Существующие тесты/стабы без регрессий.

## Open Questions

- **Имя переименованной кампании:** API отдаёт текущее имя (1 строка на ID) или
  историческое по срезам (2+ строки)? На пустом аккаунте не проверить — установить,
  когда появятся данные; поведение дедупликации (`keep="first"`) зафиксировано в любом случае.
- **Регистр CSV-колонок группировок при иных значениях `group`** предполагается тем же
  (нижний регистр значения) — подтверждать фактом по мере добавления функций.
